# Asking a human

Rules for any step whose job is to draw something out of a person — an idea, a
requirement, a decision. They govern skill-writing's own interrogation, and they
are written into any skill it authors that interviews a user, because whatever
skill-writing models, the skills it writes reproduce.

## Open on the fork

Ask first the question whose answer collapses the most other questions. Not the
most obvious one, not the first one in the list.

A menu of parallel options settles nothing: the user picks one, and every question
that was open is still open. A fork closes a dozen of them in a single answer —
whether the thing is a private tool or a public product decides accounts, data
sources, cost ceilings, and half the UI, and none of those need asking once it is
answered.

Order the questions by what they eliminate, and ask the top one.

## Recommend a claim, and give its reason

A neutral label gives the user nothing to push against. An argued position does:

> "I recommend A. The pain here is choice overload, not missing information —
> Maps already saturates the information and that is what makes deciding hard."

The user can now disagree with the *reason*, and that is where the truth arrives.
Recommending without a reason is a coin toss wearing a suit.

## Say what the question decides

Lead with what changes depending on the answer, not with the options. A question
that carries its own stakes surfaces a misunderstanding on the first turn instead
of the third — "this is about the skill's own name, not the file it writes" is one
clause, and it is the difference between an answer and a wasted round.

## Name a contradiction on the spot

When an answer contradicts an earlier one, say so and ask which holds. Agreeing
with both sides of an inconsistency defers the collision to the moment it is
expensive.

## Bounds

- One question at a time.
- Only where the answer would change the output; when nothing further would, stop.
- Infer what can be inferred. A question the user cannot answer better than you
  can is a question you should not have asked.
